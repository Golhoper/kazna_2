# Казна 2.0

## MVP 1
1) Одна Заявка (оплата поставщику)
2) Предмет оплаты 
3) Платежная позиция
4) Согласование Заявки

## MVP 2
1) Реестр
2) Платежная позиция
3) Согласовывать реестр


Что надо сделать

-Заявка создается и данные заполняются в ней вручную
-Предмет оплаты (ПО) создается и заполняется вручную. 
у заявки есть связь с предметом оплаты. 
У одной заявки может быть несколько предметов оплаты. 
ПО можно редактировать и дублировать.
-П-позиция (ПП) создается автоматически на основе данных предмета оплаты после отправки заявки на согласование. 
ПП может состоять из нескольких предметов оплаты. 
ПП можно редактировать
-Реестр платежей (РП) создается автоматически после согласования заявки. 
РП может содержать несколько ПП. 
РП можно согласовать
